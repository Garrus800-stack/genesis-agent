## [7.9.22]

The v7.9.20 self-improvement loop's first long autonomous field run surfaced a set of fidelity and tuning issues spread across the verification the loop trusts, the metrics it reports, the idle inspection loop, and the plan-to-goal linkage. This release grounds the loop's verification in evidence the system actually produced rather than the model's own assertions, settles a cluster of event, goal, and model fidelity points, hardens the test harness against leaked temporary state, tunes the live-run metrics and gates so a long solo run is neither silenced nor misread, and closes two idle-loop leaks that kept re-proposing work already done. Every change is against the runtime trace and at the root; none touches the kernel or the gate set.

### The autonomy loop verifies against evidence, not its own narration

The agent loop's analyze step fed the model a blind head-slice of a file and trusted the model's prose verdict on it, and recovery then counted any "verified" tag the model emitted as a real pass. Three changes ground this. The analyze step now sources the real module text into the prompt through a shared budget helper — a generous window for analysis, a tighter one for code generation — so the model reasons over the actual code rather than its opening lines. A syntax verdict from the real parser is computed before the model speaks and recorded on the stored insight as a verified fact (true, false, or null when unverifiable), kept separate from the model's unverified opinion. And recovery's confidence gate now turns on programmatic passes — work the system ran and checked — instead of the count of self-labelled "verified" tags, which a degraded backend could mint for free. A duplicate goal-verification routine and its only-caller tag extractor, left in the steps module after an earlier split, are removed so the recovery copy is the single path.

### A cluster of event, goal, and model fidelity fixes

Several signals the loop reads or emits carried small untruths. A goal lookup that scanned by description now resolves by id, and a step whose target carries a glob wildcard no longer demands a concrete file. The event store's replay and integrity walkers now carry a crash-restart fork — a reused event id whose dead branch is superseded by the live sibling the chain runs on from — and an exact duplicate line, so neither corrupts the rebuilt state nor fails the integrity check, while real tamper still fails both. A model-name fallback fills in the called model when the backend returns none, instead of recording "unknown". The goal-terminal events settle on a single id field, dropping a duplicate the schemas carried. Idle-mind's plan store learns the goal-to-plan link forward, at the moment the plan spawns its goal, and flips the plan's status when that goal completes, fails, or is abandoned; the module is split along journal, plan, and status lines to stay under the file-size guard. Homeostasis tracks the knowledge-graph node count from the live graph rather than a stale seed. A dedicated intent pattern recognises an explicit agent-goal phrasing. Chat, agent-loop, self-modification, and shell outcomes now carry the backend that produced them. And the trust levels gain a name-by-number reverse map, so the status and the reason strings read in words rather than integers.

### Test-harness hygiene

The test harness left its scratch directories behind — every run littered the system temporary folder with per-run roots. The harness now registers each root it creates and removes them on exit, and the leak sites across the suite are routed through that registered-root helper. A ratchet contract forbids a test from writing to a hard-coded absolute temporary path, so the leak cannot silently return. An archive-path ternary in the memory consolidator that produced the same value on both branches collapses to the single expression it always yielded.

### Tuning from the live run

The field run showed several gates and metrics reading the wrong thing. A try-guarded missing dependency — a require with a working fallback — is now ranked informational rather than high severity, decided by parsing the module and checking whether the require sits inside a try block; an unguarded missing require keeps its high signal, and on a parse failure the severity stays high conservatively. The fitness activity counters, which a crash inside the debounce window could leave under-counting, are reconciled on boot against the archived completions and the durable chat log — only the entries that post-date the last evaluation, so a mature instance does not re-fire the milestone on every restart. The self-verification calibration score now weights each prediction's error by the confidence the system actually claimed, so a cold-start guess no longer drags the metric toward chance as hard as a confident call. The symbolic guidance gate adds a breadth-scaled relevance floor: a genuinely general lesson must score a distinctly higher relevance for the step at hand before it guides, which ends the monoculture where one over-general lesson fired on essentially every step, while a category-specific lesson is unaffected and a saturated lesson is never permanently silenced. And the proactive self-expression gate relaxes its score bar in extended solo operation — past an hour with no user, measured from process boot when no user message is on record — so a long run alone is not silenced, while the rate guard stays intact against bursts.

### The idle inspection loop stops re-proposing finished work

The autonomous idle loop kept re-inspecting modules it had already covered. Two leaks fed it. A stored analysis is now attributed to a module only when the target names a module the self-model actually knows, compared path-normalised so the membership holds on every platform; a targetless analysis and an off-list path — a snapshot copy the model named off the candidate list — are no longer recorded as phantom coverage. And the file ordering the planner receives becomes a structural constraint: when enough uncovered modules exist to fill the offered list, it is built from uncovered files alone, so the planner cannot be handed a file it has already inspected, with the covered-last ordering kept as the fallback so the loop never starves.

### A pre-link plan is reconciled against its terminal goal

A plan whose goal-to-plan link predates the forward link — or was lost to a crash between a goal's creation and its terminal event — strands at its initial status while its goal sits archived as terminal. A standing, idempotent boot pass heals it: for a plan with no link and a non-terminal status, it looks in the goal archive for exactly one terminal goal whose description matches the plan's title (the title is the goal's description by construction) and adopts that goal's id and terminal status; an ambiguous, already-linked, already-terminal, or unmatched plan is left untouched.

### Notes

- Test files: 552 → 573 — coverage for each change: the grounding helpers and the recovery gate; the event-store fork-and-duplicate guard; the goal-by-id lookup and wildcard-target guard; the model-name fallback; the goal-event id field; the idle-mind plan link and the module split; the homeostasis node count; the agent-goal intent pattern; the outcome backend attribution; the trust-level names; the harness self-cleanup and the absolute-temporary-path ratchet; the dependency-severity, counter-reconcile, calibration-weight, relevance-floor, and solo-relaxation tuning; the coverage-attribution guard and the uncovered-only ordering; and the pre-link plan reconcile.
- This release changes runtime behaviour across the autonomy layer. The loop reasons over real source and gates recovery on work it ran; the event store survives a crash-restart fork; a model name and an outcome's backend are recorded from the live values; idle-mind links plans to goals both forward and, on boot, backward; the fitness counters survive a crash window; the calibration, guidance, and proactive gates read the field-measured signal instead of noise; and the idle inspection loop is offered only uncovered files. No new event type, schema, or service is added; the new modules are the grounding helper, the agent-goal pattern, and the three idle-mind split files, and one parser is exported for the dependency-severity check.
